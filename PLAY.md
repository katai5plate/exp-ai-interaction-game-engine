# テストプレイヤーガイド

このドキュメントは、AIがこのゲームエンジンでテストプレイを行うための手順書です。

## あなたの役割

あなたはテストプレイヤーです。コマンドラインでゲームを進め、プレイした感想・気づき・改善提案を開発者にフィードバックしてください。

## ゲームの始め方

```
node new-game.js
```

現在のシーンの説明と選択肢が表示されます。

## ゲームの進め方

```
node interaction.js {選択肢の番号}
```

例：選択肢 `[1]` を選ぶ場合

```
node interaction.js 1
```

次のシーンの説明と選択肢が表示されます。選択肢が表示されなくなったらゲーム終了です。

## 出力の読み方

```
光が見えるが、川が立ちはだかっている。  ← 現在のシーンの説明
[1] ボートで川を渡る                    ← 選択肢（番号で選ぶ）
[2] 引き返す
```

- 説明文はあなたの現在の状況を表します
- 選択肢はあなたのインベントリ（持ち物）によって変わることがあります
- 選択肢が表示されない場合、そのシーンはゲームの終端（クリアまたはゲームオーバー）です

## テストプレイの進め方

1. **まず通しでプレイする** — 自然な判断で一度エンディングまで（またはゲームオーバーまで）プレイしてください
2. **別ルートを試す** — 前回と異なる選択肢を選んで、分岐を探索してください
3. **フィードバックをまとめる** — プレイを終えたら以下の観点で感想をまとめてください

## フィードバックの観点

- **理解しやすさ**：説明文を読んで状況が把握できたか？選択肢の意味は明確だったか？
- **納得感**：選択の結果は期待通りだったか？理不尽に感じた場面はあったか？
- **発見の楽しさ**：新しいルートや選択肢を見つけたとき、何か感じたか？
- **バランス**：詰まった場面はあったか？簡単すぎた・難しすぎた箇所はあったか？
- **気になった点**：違和感のある表現、ミス、挙動の不自然さ

## ゲームをやり直すには

```
node new-game.js
```

いつでも実行すると状態がリセットされ、最初からプレイできます。
